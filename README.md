El proyecto se construyó aplicando los principios de la programación orientada a objetos: cada entidad relevante del dominio del torneo (equipo, partido, torneo) se modela como una clase independiente, con sus atributos encapsulados mediante campos privados y expuestos de forma controlada a través de propiedades públicas. La interfaz gráfica (Form1) se mantiene separada de esta lógica de negocio y se limita a capturar la interacción del usuario, delegar las operaciones a los objetos del dominio y refrescar la información visible en pantalla.
